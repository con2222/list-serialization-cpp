# Сериализация и десериализация двусвязного списка

Программа представляет собой консольную утилиту для сериализации и десериализации двусвязного списка в бинарный файл. Особенность структуры в том, что каждый узел содержит случайный указатель (`rand`) на любой другой узел этого же списка.

## Архитектура проекта
Проект спроектирован с учетом принципа единственной ответственности (SRP) и разделен на логические модули:
* **`List`** — сама структура данных и логика сериализации/десериализации.
* **`ArgParser`** — обработка флагов командной строки.
* **`FileParser`** — парсинг текстовых файлов и первичное заполнение списка.

## Сборка проекта (CMake)

Проект использует CMake для кроссплатформенной сборки. Для компиляции (Linux / macOS / Windows) выполните следующие команды из корня проекта:

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

## Использование и флаги (CLI)

Программа поддерживает гибкую настройку через аргументы командной строки:
* `-h` или `--help` — вывести справку по использованию.
* `-i <файл>` — указать входной текстовый файл для чтения списка (по умолчанию: `inlet.in`).
* `-o <файл>` — указать выходной бинарный файл для сериализации (по умолчанию: `outlet.in`).

**Режим проверки**
* `-b <входной_бинарник> <выходной_текст>` — эксклюзивный флаг десериализации. Программа прочитает указанный бинарный файл, восстановит список в памяти и сохранит его обратно в текстовом формате для удобной сверки данных.

**Примеры запуска:**
```bash
./list-serialization -i my_data.txt -o list.bin
./list-serialization -b list.bin result.txt
```

## Формат данных

### Текстовый формат (Входные данные / Вывод флага -b)
Каждая строка описывает один узел в формате `<data>;<rand_index>`, где `<rand_index>` — это порядковый номер узла, на который ссылается `rand` (начиная с 0), либо `-1`, если указатель пустой.

### Бинарный формат (Сериализация)
Структура записи каждого узла идет строго последовательно:
1. `uint32_t` (4 байта) — длина строки `data`.
2. `char*` (N байт) — содержимое строки (без нуль-терминатора).
3. `int32_t` (4 байта) — индекс узла, на который ссылается `rand` (-1, если пустой).

## Ключевые алгоритмы
* **`serialize_list`**: Проходит по списку и сохраняет состояние каждого узла в бинарный файл. Для перевода указателей в индексы используется `std::unordered_map`, что обеспечивает общую сложность **O(N)**.
* **`deserialize_list`**: Считывает бинарный файл, восстанавливает структуру списка и реконструирует связи `rand` по сохраненным индексам. Выполняется за один цикл чтения и один цикл связывания (**O(N)**).
* **`save_to_text`**: Вспомогательный метод для проверки целостности данных после десериализации.
* **`clear_list`**: Гарантирует корректное и полное освобождение выделенной памяти в деструкторе, предотвращая утечки.
