# Сериализация и десериализация двусвязного списка

Программа представляет собой консольную утилиту для сериализации и десериализации двусвязного списка в бинарный файл. Особенность структуры в том, что каждый узел содержит случайный указатель (`rand`) на любой другой узел этого же списка.

## Требования
* Компилятор с поддержкой **C++17**.
* **CMake** версии 3.1.2 или выше.

## Архитектура проекта
Программа разделена на логические модули:
* **`List`** — структура данных и логика сериализации/десериализации.
* **`ArgParser`** — обработка флагов командной строки.
* **`FileParser`** — парсинг текстовых файлов и первичное заполнение списка.

## Сборка проекта (CMake)

Программа использует CMake для кроссплатформенной сборки. Для компиляции (Linux / macOS / Windows) выполните следующие команды из корня проекта:

```bash
cmake -S . -B build
cmake --build build
```

## Использование и флаги (CLI)

Программа поддерживает гибкую настройку через аргументы командной строки:
* `-h` или `--help` — вывести справку по использованию.
* `-i <файл>` — указать входной текстовый файл для чтения списка (по умолчанию: `inlet.in`).
* `-o <файл>` — указать выходной бинарный файл для сериализации (по умолчанию: `outlet.out`).

**Режим десериализации и проверки**
* `-b <входной_бинарник> <выходной_текст>` — флаг десериализации. Программа прочитает бинарный файл, восстановит список в памяти и сохранит его обратно в текстовом формате для удобной сверки данных.

**Примеры запуска:**
```bash
# Сериализация из текста в бинарник
./list-serialization -i my_data.txt -o list.bin

# Десериализация из бинарника в текст для проверки
./list-serialization -b list.bin result.txt
```

## Формат данных

### Текстовый формат (Входные данные / Вывод флага -b)
Каждая строка описывает один узел в формате `<data>;<rand_index>`, где `<rand_index>` — это порядковый номер узла, на который ссылается `rand` (начиная с 0), либо `-1`, если указатель пустой. Строка `<data>` может содержать любые символы.

### Бинарный формат (Сериализация)
Структура записи каждого узла идет строго последовательно:
1. `uint32_t` (4 байта) — длина строки `data`.
2. `char*` (N байт) — содержимое строки (без нуль-терминатора).
3. `int32_t` (4 байта) — индекс узла, на который ссылается `rand` (-1, если пустой).

## Ключевые алгоритмы и сложность
* **`serialize_list`**: проходит по списку и сохраняет состояние каждого узла в бинарный поток напрямую (без промежуточных буферов). Для перевода указателей в индексы используется `std::unordered_map`.
* **`deserialize_list`**: восстанавливает структуру списка и реконструирует связи `rand` по сохраненным индексам. Выполняется за один проход чтения файла и один проход связывания
* **`clear_list`**: гарантирует корректное и полное освобождение выделенной памяти в деструкторе, предотвращая утечки.

## Тестирование (Unit Tests)

Проект покрыт модульными тестами с использованием фреймворка **Google Test**. Тесты проверяют:
* Базовые операции добавления элементов.
* Граничные случаи (например, безопасная обработка пустого списка).
* Полный цикл (Round-trip): создание $\rightarrow$ сериализация $\rightarrow$ десериализация $\rightarrow$ сверка целостности данных.

Библиотека GTest подтягивается автоматически на этапе генерации проекта CMake (через `FetchContent`), ничего устанавливать вручную не нужно.

**Как запустить тесты:**
После стандартной сборки проекта, находясь в директории `build`, выполните одну из команд:

```bash
cd build
ctest
```
